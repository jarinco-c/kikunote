# きくノート - 課題管理

## 未解決の課題

### [中] Androidでの録音品質が低い
- **状況**: Androidで録音するとiPhoneと比べて文字起こし精度が低い
- **確認済み**: 端末のマイク自体の問題ではない（別アプリでは正常）
- **対応済み（効果限定的）**:
  - ビットレートを32kbps→128kbpsに引き上げ
  - sampleRate制約を削除（端末デフォルトを使用）
  - Web Audio APIでゲイン3倍増幅 + DynamicsCompressor追加
  - フォーマットをmp4(AAC)優先に変更（ただしAndroid Chromeはmp4非対応でWebM/Opusにフォールバック）
- **次回の検討事項**:
  - Web Audio APIでPCM/WAV録音する方法（MediaRecorderのエンコーダを完全にバイパス）
  - Android Chromeの MediaRecorder の既知の問題を調査

## 解決済み

### iPhone自動ロックで録音が3分で停止（2026-04-15解決）
- **症状**: iOS Safari 通常タブで録音中に放置すると、自動ロック（実測3分）で MediaRecorder が停止し、それ以降の録音が失われる。二次症状として「生成に失敗しました: Load failed」が表示される
- **原因**: Wake Lock API だけでは iOS Safari 通常タブの画面ロックを完全に抑止できない既知問題
- **対応**: AudioContext から無音を出力し続けるハック（NoSleep.js 同様のアプローチ）を Recorder.tsx に追加。silent な AudioBufferSourceNode を `audioContext.destination`（スピーカー）へループ接続し、タブを「オーディオ再生中」扱いにする
- **検証結果**: 12分録音で一度も停止せず完走（2026-04-15実機確認）
- **残課題**: 20〜30分の超長時間録音での継続確認。問題が再発した場合は PWA化（ホーム画面追加でスタンドアロン動作）や MediaRecorder.state 監視を追加検討

### 長時間録音が安定しない（2026-04-14解決）
- **旧症状**:
  - Vercel 60秒制限でセグメント文字起こしがタイムアウト
  - タイムアウトしなくても「議事録が最初の1-2分分しか生成されない」バグが並存
- **根本対応**: ホスティングをVercel→Render無料プランに移行。セグメント分割自体を廃止
  - Gemini Files API経由で単一Blobを処理（inlineDataの20MB制限を回避）
  - Render無料プランの15分スリープにはキープアライブping（10分毎）で対処
- **撤去したコード**: セグメント分割タイマー、セグメント単位のリトライ、セグメント結合時間マーカー、maxDuration=60

### 議事録が文字起こしのベタ打ちになっていた（2026-03-29解決）
- プロンプトを要約型に改善
- 文字起こしは別途DBに保存し、タブで切り替え表示可能に

### セグメント分割時のレースコンディション（2026-03-28解決）
- 10分以上の録音でセグメント分割時にデータが壊れていた
- `onstop` コールバックで停止完了を待つように修正
- **注**: 2026-04-14 にセグメント分割自体を廃止したため、このコード自体が存在しなくなった
