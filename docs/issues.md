# きくノート - 課題管理

## 未解決の課題

### [高] Androidでの録音品質が低い
- **状況**: Androidで録音するとiPhoneと比べて文字起こし精度が大幅に低い
- **確認済み**: 端末のマイク自体の問題ではない（Typeless等の別アプリでは正常）
- **対応済み（効果不十分）**:
  - ビットレートを32kbps→96kbpsに引き上げ
  - sampleRate制約を削除（端末デフォルトを使用）
- **次回の検討事項**:
  - 録音データをダウンロードして実際の音声品質を確認する
  - WebM/Opus以外のフォーマット（audio/mp4等）を試す
  - ビットレートをさらに上げる（128kbps〜）
  - MediaRecorderの代わりにWeb Audio APIでWAV録音する方法を検討
  - Android Chromeの MediaRecorder の既知の問題を調査

## 解決済み

### セグメント分割時のレースコンディション（2026-03-28解決）
- 10分以上の録音でセグメント分割時にデータが壊れていた
- `onstop` コールバックで停止完了を待つように修正
- セグメント長を10分→5分に短縮（Vercel 60秒制限対策）
- 文字起こしを並列処理化
